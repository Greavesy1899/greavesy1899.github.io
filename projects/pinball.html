PhysX Powered Pinball Machine

1 / 1
Two cities surrounded by coral, with both cities using a different tunnel layout and buildings.
❮ ❯

This pinball machine was developed for a University module named 'Physics Simulation'. The goal was to develop a pinball machine using a provided codebase and PhysX. By using PhysX, the idea was to implement key components of a pinball machine such as the plunger and the paddles, in a realistic fashion. The lecturers gave us a codebase which required quite a large amount of refactoring; originally, classes were packed into header files with little to no organisation and thus, provided me with the challenge of proving my 'refactoring' abilities. Not only did I display my 'refactoring' abilities, I also managed to find a bug and fix it - this was hidden deep in the codebase which went unnoticed for years.

To further improve my simulation, rather than using the provided code classes such as the box, spheres or capsules to build the machine, I implemented my own object importer to provide the ability to build my own machine in Blender. This allowed me to import custom Wavefront .OBJ files and 'cook' them ready for PhysX. As well as having basic support for this, I went a step further to provide material indices for PhysX; during the development of this system I noticed the bug and went onto fix it. As well as finding ways to improve functionality, I also searched the API for certain joints such as the PxD6Joint, and other functionality like Continous Collision Detection (CCD).

Developed Skills

  • Better Knowledge of the PhysX SDK.
  • Improved my skills in using and refactoring existing codebases.
  • Improved my skills in general research, such as looking for documentation in the API.
  • Developed an understanding of certain techniques such as CCD.
